# Starmap - Техническое задание

## Обзор проекта
Starmap — корпоративное веб-приложение для управления техническими компетенциями команды, построенное на Rails 8.1.1 с использованием Hotwire для интерактивности. Цель проекта — трансформировать управление скрытыми знаниями в проактивную деятельность, обеспечивая прозрачность и устойчивость команды.

## Основные требования

### Функциональные требования

#### Система управления компетенциями
- **Шкала оценок**: 0-3 (0=не имею представления, 1=имею представление, 2=свободно владею, 3=могу учить других)
- **Процесс утверждения**: Самооценка разработчика → утверждение тимлидом
- **История изменений**: Отслеживание изменений по кварталам
- **Валидации**: Ограничения на основе ролей и статуса квартала

#### Квартальные циклы
- **Статусы кварталов**: draft → active → closed → archived
- **Автоматическое создание**: Новые кварталы с копированием предыдущих оценок
- **Ограничения редактирования**: Только в статусе draft/active
- **История**: Полная история изменений компетенций

#### Три типа дашбордов
- **Overview Dashboard**: Общие метрики и риски для unit lead/CTO
- **Team Dashboard**: Детальный анализ компетенций команды для тимлидов
- **Personal Dashboard**: Персональный прогресс для разработчиков

#### Планы развития
- **Создание планов**: На основе пробелов в компетенциях
- **Статусы**: active → completed/paused
- **Привязка к кварталам**: Целевые кварталы для достижения целей

#### LDAP интеграция
- **Синхронизация пользователей**: Автоматическая каждые 24 часа
- **Аутентификация**: Через корпоративный LDAP
- **Роли и группы**: Маппинг LDAP групп на роли системы

### Нефункциональные требования

#### Безопасность
- **Аутентификация**: Devise + LDAP integration
- **Авторизация**: Pundit с четырёхуровневой ролевой системой
- **Защита данных**: CSRF, XSS, SQL injection protection
- **Логирование**: Структурированные логи с фильтрацией чувствительных данных

#### Производительность
- **Время отклика**: < 2 секунд для основных операций
- **Масштабирование**: Поддержка до 500 пользователей
- **Кеширование**: Solid Cache для метрик и LDAP данных
- **Оптимизация запросов**: Eager loading, индексы, оптимизированные scopes

#### Надёжность
- **Доступность**: 99.5% uptime
- **Резервное копирование**: Автоматические бэкапы PostgreSQL
- **Мониторинг**: Health checks, мониторинг фоновых задач
- **Восстановление**: Disaster recovery план

### Архитектурные требования

#### Технологический стек
- **Backend**: Ruby on Rails 8.1.1
- **Frontend**: Hotwire (Turbo + Stimulus)
- **База данных**: PostgreSQL 15+
- **Очереди**: Solid Queue (встроенная в Rails)
- **Кеширование**: Solid Cache
- **Аудит**: Audited gem для отслеживания изменений

#### Интеграции
- **LDAP**: Net-LDAP для корпоративной аутентификации
- **Email**: Letter Opener (development), SMTP (production)
- **Мониторинг**: Встроенные Rails health checks

#### Развёртывание
- **Контейнеризация**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **Окружения**: Development, Test, Production
- **Конфигурация**: Environment variables + YAML файлы

## Пользовательские роли и права доступа

### Engineer (Разработчик)
- **Просмотр**: Personal dashboard, собственные оценки
- **Редактирование**: Собственные оценки в черновиках/активных кварталах
- **Создание**: Личные планы развития
- **Ограничения**: Нет доступа к чужим данным

### Team Lead (Тимлид)
- **Все права Engineer +**
- **Просмотр**: Team dashboard, оценки всей команды
- **Утверждение**: Оценки подчинённых
- **Создание**: Командные планы развития
- **Ограничения**: Только своя команда

### Unit Lead
- **Все права Team Lead +**
- **Просмотр**: Overview dashboard, все команды юнита
- **Управление**: Юнитом, распределение ресурсов
- **Стратегическое планирование**: Долгосрочные цели развития

### Admin (Администратор)
- **Полные права**: На все ресурсы системы
- **Управление**: Пользователи, технологии, кварталы
- **Системные настройки**: Конфигурация, интеграции
- **Аудит**: Полный доступ к логам и истории

## Ключевые бизнес-метрики

### Coverage Index
- **Определение**: Процент технологий с ≥2 экспертами (рейтинг 2-3)
- **Цель**: >80% для стабильной команды
- **Критические технологии**: Требуют минимум 2 экспертов

### Maturity Index
- **Определение**: Средняя оценка по всем технологиям
- **Шкала**: 0.0 - 3.0
- **Цель**: >2.0 для зрелой команды

### Red Zones
- **Определение**: Критические технологии с недостаточным покрытием
- **Критерии**: Критичность=high + <2 экспертов
- **Действие**: Приоритетное обучение команды

### Key Person Risk
- **Определение**: Технологии где один сотрудник единственный эксперт
- **Метрика**: Количество критических технологий в зоне риска
- **Mitigation**: План передачи знаний

## Технические ограничения

### Ограничения платформы
- **Ruby**: MRI 3.2+
- **PostgreSQL**: Версия 15+
- **Браузеры**: Chrome 90+, Firefox 88+, Safari 14+
- **Мобильные**: Responsive design (iOS Safari, Android Chrome)

### Ограничения производительности
- **Пользователи**: До 500 активных пользователей одновременно
- **Данные**: До 50 технологий, 1000 пользователей, 10 команд
- **Запросы**: Время генерации страницы < 2 секунд
- **Фоновые задачи**: LDAP sync не более 5 минут

### Ограничения безопасности
- **Сессии**: Максимум 8 часов бездействия
- **Пароли**: Минимум 8 символов, проверка сложности
- **LDAP**: Только SSL/TLS соединения
- **Логи**: Автоматическое исключение чувствительных данных
