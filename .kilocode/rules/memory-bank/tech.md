# Технологии Starmap

## Основной стек технологий

### Backend/Frontend Framework
- **Ruby on Rails 8.1.1** - Основной фреймворк приложения
- **Hotwire (Turbo + Stimulus)** - Для интерактивного UI без отдельного frontend фреймворка
  - **Turbo**: Ускоряет навигацию и обновления страниц
  - **Stimulus**: Контроллеры для интерактивности

### База данных
- **PostgreSQL 1.1** - Основная база данных
- **pg gem** - Адаптер для PostgreSQL

### Веб-сервер
- **Puma** - Concurrent web server (версия >= 5.0)

### Аутентификация и авторизация
- **Devise** - Основная система аутентификации
- **devise_ldap_authenticatable** - Интеграция с LDAP для корпоративной аутентификации
- **Pundit** - Система авторизации на основе политик

### Фоновые задачи и кеширование
- **Solid Queue** - Система управления очередями фоновых задач (замена Redis/Sidekiq)
- **Solid Cache** - Система кеширования данных

### Аудит и версионирование
- **Audited** - Отслеживание изменений в моделях и данных

### Интеграции
- **net-ldap** - Ruby библиотека для работы с LDAP
- **letter_opener** - Открытие email в браузере (только для development)

### Разработка и тестирование
- **rspec-rails** - Framework для тестирования
- **factory_bot_rails** - Генерация тестовых данных
- **shoulda-matchers** - Упрощенные матчеры для тестов
- **capybara** - Integration тестирование
- **selenium-webdriver** - Автоматизация браузера для тестов
- **database_cleaner-active_record** - Очистка базы данных между тестами
- **debug** - Отладчик Ruby (development/test)

### Безопасность и анализ
- **brakeman** - Статический анализ безопасности Rails приложения
- **annotate** - Автоматическое добавление комментариев к моделям (development)

## Системные зависимости

### Дата и время
- **tzinfo-data** - Данные временных зон (для Windows и JRuby)

### Конфигурация

#### Основные файлы конфигурации:
- `config/application.rb` - Основная конфигурация Rails приложения
- `config/database.yml` - Конфигурация базы данных PostgreSQL
- `config/ldap.yml` - Конфигурация LDAP интеграции
- `config/routes.rb` - Маршруты приложения

#### Environment-specific конфигурации:
- `config/environments/development.rb` - Настройки для development
- `config/environments/test.rb` - Настройки для тестирования
- `config/environments/production.rb` - Настройки для production

### Локализация
- **I18n** - Встроенная система локализации Rails
- Поддержка языков: en (по умолчанию), ru
- Настройка в `config/application.rb`

## Архитектурные решения

### Выбор технологий
1. **Rails 8.1.1**: Последняя версия с улучшенной производительностью
2. **Hotwire**: Современный подход к интерактивности без JavaScript фреймворков
3. **Solid Queue**: Встроенная система очередей Rails, замена внешних решений
4. **PostgreSQL**: Надежная реляционная база с расширенными возможностями

### Безопасность
- CSRF protection включен по умолчанию
- Фильтрация чувствительных параметров в логах
- Pundit для декларативной авторизации
- Devise LDAP для корпоративной аутентификации

### Производительность
- Eager loading для предотвращения N+1 запросов
- Кеширование метрик через Solid Cache
- Индексы на критических полях
- Оптимизированные scopes в моделях

### Развертывание
- **Docker/Docker Compose** - Контейнеризация приложения
- **Multi-stage build** - Оптимизация размера Docker образов
- Поддержка health checks для мониторинга

## Разработка

### Команды для разработки
```bash
# Запуск сервера разработки
bin/dev

# Выполнение миграций
bin/rails db:migrate

# Запуск тестов
bin/rails test

# Консоль Rails
bin/rails console
```

### Среда разработки
- **Локализация**: Временная зона Europe/Moscow
- **Логирование**: Структурированные логи
- **Отладка**: debug gem для отладки кода
- **Тестирование**: Полный стек тестирования с RSpec

### Инструменты качества кода
- **Brakeman**: Статический анализ безопасности
- **Annotate**: Автоматическая документация моделей
- **Letter Opener**: Просмотр email в development

## Ограничения и особенности

### Платформы
- Поддержка MRI Ruby на всех платформах
- Windows поддержка через tzinfo-data
- JRuby поддержка с ограничениями

### Производительность
- Размер кэша настраивается через Solid Cache
- Ограничения concurrent connections через Puma
- Оптимизация для средних и малых команд (до 500 пользователей)

### Безопасность
- Зависимость от корпоративного LDAP
- Требования к SSL/TLS для production
- Необходимость настройки CORS для API интеграций
