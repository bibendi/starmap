# Implementation Tasks: Admin Quarter Management

## Technical Setup (Foundation)

- [x] 1. Настроить административный namespace и маршрутизацию
  Создать namespace `/admin` в маршрутизации для изоляции административных страниц. Реализовать базовую структуру контроллера и убедиться в доступе для пользователей с ролью admin.
  _Requirements: 1.4, 1.5_

## Parallel Development Tasks

- [x] 2. Реализовать сервис для управления жизненным циклом квартала (P)
  - [x] 2.1 Создать сервис для переходов между статусами квартала
    Реализовать методы активации, закрытия и архивирования квартала. Добавить проверки предусловий для каждого перехода (draft→active, active→closed, closed→archived). Использовать транзакции для обеспечения атомарности операций.
    _Requirements: 3.1, 3.3, 3.4_

  - [x] 2.2 Добавить валидацию бизнес-правил при смене статусов
    Реализовать проверку наличия активного квартала при активации нового. Добавить проверку связанных данных перед удалением. Обеспечить корректную обработку ошибок с понятными сообщениями.
    _Requirements: 3.5, 5.2_

  - [x] 2.3 Реализовать копирование данных из предыдущего квартала
    Создать логику копирования рейтингов компетенций из предыдущего закрытого квартала при активации нового. Обрабатывать случай отсутствия предыдущего квартала.
    _Requirements: 3.1_

  - [x] 2.4 Протестировать сервис управления статусами
    Написать юнит-тесты для всех методов сервиса: успешные переходы, ошибочные переходы, копирование данных, валидации бизнес-правил.
    _Requirements: 3.1, 3.3, 3.4, 3.5_

- [x] 3. Реализовать отображение списка кварталов (P)
  - [x] 3.1 Создать страницу со списком всех кварталов
    Реализовать табличное отображение кварталов с постраничной разбивкой (25 элементов на страницу). Добавить сортировку по году и номеру квартала (по убыванию).
    _Requirements: 1.1, 1.3_

  - [x] 3.2 Добавить отображение метаданных квартала в списке
    Отображать название квартала, статус, даты начала/окончания, признак текущего квартала. Добавить ссылки на просмотр деталей каждого квартала.
    _Requirements: 4.1_

  - [x] 3.3 Добавить фильтрацию по статусу квартала
    Реализовать фильтр по статусам: draft, active, closed, archived. Добавить возможность сброса фильтра для отображения всех кварталов.
    _Requirements: 1.3_

  - [x] 3.4 Добавить кнопки действий для каждого квартала
    Реализовать кнопки переходов между статусами (активировать, закрыть, архивировать) с проверкой прав доступа. Добавить кнопку удаления для черновиков с подтверждением.
    _Requirements: 1.2, 3.1, 3.3, 3.4, 5.1_

- [ ] 4. Реализовать создание нового квартала (P)
  - [ ] 4.1 Создать форму для создания квартала
    Реализовать форму с полями: название, год, номер квартала (1-4), даты начала/окончания, даты периода оценки. Добавить авторасчет дат на основе года и номера квартала.

  - [ ] 4.2 Добавить валидацию данных при создании
    Реализовать проверку уникальности названия в рамках года. Добавить валидацию отсутствия пересечений с существующими кварталами. Проверить корректность диапазонов дат (начало < конец).

  - [ ]* 4.3 Протестировать создание квартала
    Написать тесты на успешное создание, валидацию дубликатов, валидацию дат, отображение ошибок в форме.
    _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5_
