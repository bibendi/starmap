# Feature Specification: Team Skill Matrix Redesign

**Feature Branch**: `005-team-skill-matrix-redesign`  
**Created**: 2026-04-11  
**Status**: Draft  
**Input**: User description: "Переделываем TeamSkillMatrixComponent — переименовать заголовок, убрать колонки участников, добавить Coverage progress bar, создать отдельную страницу технологии команды с таблицей участников, проверить видимость меню Команда для unit_lead"

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Обновлённая матрица компетенций команды (Priority: P1)

Тимлид или юнит-лид открывает страницу команды и видит переименованную карточку «Компетенции команды» (вместо «Карта навыков команды»). Таблица содержит только технологии и новые метрики — без колонок отдельных участников. Для каждой технологии отображается Bus Factor (как раньше) и новая колонка Coverage в виде progress bar, показывающая процент технологий с достаточным покрытием экспертами.

**Why this priority**: Это основное изменение интерфейса — новая таблица заменяет текущую и является точкой входа для навигации к деталям.

**Independent Test**: Открыть страницу любой команды и убедиться, что таблица содержит колонки: Технология, Bus Factor, Coverage — и не содержит колонок с именами участников. Progress bar корректно отображает процент покрытия.

**Acceptance Scenarios**:

1. **Given** пользователь с ролью тимлид/юнит-лид/админ, **When** открывает страницу команды, **Then** видит карточку с заголовком «Компетенции команды»
2. **Given** таблица компетенций, **When** просматривает колонки, **Then** видит: Технология (с категорией, критичностью), Bus Factor, Coverage — без колонок участников
3. **Given** технология с expert count = 2 и target_experts = 3, **When** отображается Coverage, **Then** progress bar показывает 67%
4. **Given** технология с target_experts достигнутым, **When** отображается Coverage, **Then** progress bar показывает 100%
5. **Given** технология без единого эксперта (0 из 2), **When** отображается Coverage, **Then** progress bar показывает 0%

### User Story 2 — Страница технологии команды с участниками (Priority: P1)

Пользователь кликает на название технологии в таблице компетенций и переходит на отдельную страницу `/teams/:team_id/technologies/:id`. На странице отображается: заголовок с названием технологии, под заголовком — кликабельное название команды (ведёт на страницу команды), и таблица участников команды с их рейтингами по данной технологии в текущем квартале.

**Why this priority**: Это ключевая навигация — детали по участникам, убранные из матрицы, теперь доступны через отдельную страницу.

**Independent Test**: Кликнуть на любую технологию в матрице → перейти на страницу технологии → увидеть таблицу участников с рейтингами и кликабельное название команды.

**Acceptance Scenarios**:

1. **Given** таблица компетенций на странице команды, **When** кликает на название технологии, **Then** переходит на страницу `/teams/:team_id/technologies/:technology_id`
2. **Given** страница технологии команды, **When** страница загружена, **Then** под основным заголовком отображается кликабельное название команды, ведущее на страницу команды
3. **Given** страница технологии команды, **When** просматривает таблицу, **Then** видит список участников с их рейтингами (0–3) по данной технологии
4. **Given** участник без рейтинга по технологии, **When** отображается таблица, **Then** рейтинг показан как «—» или 0
5. **Given** страница технологии команды, **When** просматривает рейтинг участника, **Then** видит цветовую индикацию (Expert/Proficient/Basic/No knowledge) как в текущей матрице

### User Story 3 — Видимость меню «Команда» для юнит-лида (Priority: P2)

Юнит-лид авторизуется в системе и видит пункт меню «Команда» в навигации. Сейчас для юнит-лида этот пункт может быть скрыт из-за проверки `current_user.team` — юнит-лид не всегда привязан к конкретной команде.

**Why this priority**: Это баг, блокирующий навигацию для целевой роли. Без исправления юнит-лид не может полноценно использовать интерфейс.

**Independent Test**: Зайти под юнит-лидом и убедиться, что пункт «Команда» виден в навигации и ведёт на корректную страницу.

**Acceptance Scenarios**:

1. **Given** пользователь с ролью unit_lead без привязки к конкретной команде, **When** открывает любую страницу, **Then** видит пункт меню «Команда»
2. **Given** юнит-лид без привязки к конкретной команде кликает «Команда», **When** переходит по ссылке, **Then** попадает на страницу `/teams` (список команд его юнита)

---

### Edge Cases

- Технология добавлена в команду, но ни один участник не имеет рейтингов — Coverage = 0%, таблица участников показывает всех с пустыми рейтингами
- Команда без технологий — отображается пустое состояние матрицы
- Квартал ещё не начат (draft) — рейтинги не отображаются или показываются как «—»
- Удалённый участник (покинувший команду) — его рейтинги не показываются в таблице на странице технологии
- Несколько команд в юните у юнит-лида — навигация «Команда» ведёт на `/teams` (список команд юнита)

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Заголовок карточки матрицы навыков MUST отображаться как «Комплекты команды» (локализованный ключ «Компетенции команды»)
- **FR-002**: Таблица в компоненте компетенций команды MUST содержать только колонки: Технология (с категорией и критичностью), Bus Factor, Coverage
- **FR-003**: Колонки с именами участников MUST быть убраны из компонента матрицы компетенций
- **FR-004**: Колонка Coverage MUST отображать процентное соотношение (количество экспертов / target_experts × 100%) в виде progress bar
- **FR-005**: Progress bar Coverage MUST визуально отражать уровень покрытия: низкий (красный/опасный), средний (жёлтый/предупреждение), высокий (зелёный/успех)
- **FR-006**: Каждое название технологии в таблице MUST быть кликабельной ссылкой на страницу `/teams/:team_id/technologies/:technology_id`
- **FR-007**: Страница `/teams/:team_id/technologies/:technology_id` MUST отображать таблицу участников команды с их рейтингами по данной технологии в текущем квартале
- **FR-008**: Под заголовком страницы технологии MUST отображаться кликабельное название команды, ведущее на `/teams/:team_id`
- **FR-009**: Доступ к странице технологии команды MUST контролироваться авторизацией (тимлид, юнит-лид, админ — как для страницы команды)
- **FR-010**: Пункт меню «Команда» MUST быть видим для роли unit_lead даже без привязки к конкретной команде
- **FR-011**: Таблица участников на странице технологии MUST использовать существующую цветовую индикацию рейтингов (0–3)

### Key Entities

- **TeamTechnology**: Связь команды и технологии с атрибутами criticality, target_experts — используется для расчёта Coverage
- **SkillRating**: Рейтинг участника (0–3) по технологии в конкретном квартале — отображается в таблице участников на странице технологии
- **Coverage**: Производная метрика — отношение количества экспертов (rating ≥ 2) к target_experts для конкретной технологии в команде

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователь видит переименованную карточку «Компетенции команды» без колонок участников — таблица загружается менее чем за 1 секунду
- **SC-002**: Coverage корректно отображается для каждой технологии с визуальным progress bar — значение совпадает с расчётом (эксперты / target × 100%)
- **SC-003**: Клик по технологии ведёт на страницу с таблицей участников — навигация занимает не более 1 секунды
- **SC-004**: Юнит-лид видит пункт «Команда» в навигации без дополнительных настроек профиля
- **SC-005**: Существующий функционал Bus Factor не нарушен — значения идентичны до и после изменений

## Clarifications

## Assumptions

- Coverage для отдельной технологии = (количество участников с рейтингом 2–3 / target_experts) × 100%, аналогично существующему Coverage Index, но на уровне одной технологии
- Страница `/teams/:team_id/technologies/:id` доступна всем ролям, имеющим доступ к просмотру команды (тимлид своей команды, юнит-лид команд своего юнита, админ — всех команд)
- Цветовая шкала progress bar: 0–49% — красный, 50–79% — жёлтый, 80–100% — зелёный (по аналогии с существующими risk level для Bus Factor)
- Рейтинги на странице технологии показываются за текущий активный квартал
- Таблица участников на странице технологии не требует отдельного ViewComponent — рендерится напрямую в шаблоне
- Для юнит-лида без привязки к конкретной команде навигация «Команда» ведёт на существующую страницу `/teams` (список команд юнита)
- Используется существующая система авторизации Pundit с расширением NavigationPolicy
- Легенда рейтингов (0–3) переносится на страницу технологии для контекста
